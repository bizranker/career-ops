# Why Not Post the Old Kubernetes Cluster Creation Script

The older cluster creation script had interesting historical value but was not prioritized for the public portfolio because:

- modern teams do not usually use one-size-fits-all Bash-driven cluster bootstrap flows as best practice
- it risks signaling outdated cluster automation patterns
- stronger repos already exist that demonstrate more timeless engineering value

The better public story is:
- infrastructure automation
- observability
- Linux provisioning
- drift detection
- AWS/Terraform lab work
